## 背景與目的

飼主在體驗過優質的保母服務後，通常會有長期合作的傾向。為了提升飼主的回訪率並簡化再次預約的流程，系統需提供「我的最愛」收藏功能。飼主可以集中管理心儀的保母清單，並即時掌握保母的服務狀態（是否開放預約）。

## 使用者故事

- 身為 **飼主**，當我與某位保母完成交易後，我希望能一鍵將其加入「我的最愛」，方便下次找他。
- 身為 **飼主**，我希望能透過保母的帳號 ID 搜尋並主動將其加入收藏，即便還沒正式下單過。
- 身為 **飼主**，我希望能隨時查看我的收藏清單，並知道哪些保母目前在休息中。
- 身為 **飼主**，我希望我的收藏行為是私密的，不需要經過保母同意也不會驚動保母。

---

## 功能規格

### 主流程

**A. 加入我的最愛 (多管道)**
1. **交易後快速加入**：
   - 在訂單完成 (`COMPLETED`) 的頁面或評價引導中，顯示「收藏此保母」按鈕。
2. **主動搜尋加入**：
   - 飼主在「我的最愛」頁面點擊「新增收藏」，輸入保母的系統帳號 ID 或 Email 進行搜尋。
   - 若搜尋命中且該對象為保母角色，點擊「加入」即可完成。
3. **公開頁面加入**：
   - 在保母的公開預約網頁 (PRD-018) 點擊「♥」圖示。

**B. 我的最愛清單管理**
1. 飼主進入「個人設定 > 我的最愛保母」。
2. 清單呈現資訊：
   - 保母大頭貼、名稱、服務標籤。
   - **即時狀態標籤**：顯示「服務中」或「休息中/隱藏中」。
   - 點擊卡片可直接跳轉至該保母的預約網頁。
3. 支援「移除收藏」操作。

### 例外處理

| 情境 | 預期行為 |
|------|---------|
| 搜尋不到對應 ID | 提示「查無此保母，請確認 ID 是否正確」。 |
| 收藏已註銷的帳號 | 該保母卡片在清單中應顯示「帳號已移除」，且無法點擊進入。 |

### 資料規則 / 業務規則

- **單向隱私規則**：加入「我的最愛」是飼主的單方面行為。系統**不會**發送通知給保母，保母也無法看到誰將其加入收藏。
- **狀態連動**：
  - 若保母在 PRD-018 設定為 `Hidden`，清單中的卡片應顯示灰色調，並標註「隱藏中」。
  - 若保母在 PRD-001 將該飼主列入「黑名單」，該飼主點擊卡片進入時仍會看到「不開放預約」提示頁。
- **數量限制**：每位飼主最多可收藏 50 位保母。

---

## 驗收標準（AC）

- [ ] **AC-1 (快速加入)**：訂單完成後點擊收藏，該保母應立即出現在「我的最愛」清單中。
- [ ] **AC-2 (ID 搜尋加入)**：輸入正確的保母 ID 能成功搜尋並收藏。
- [ ] **AC-3 (狀態連動驗證)**：當保母切換為 Hidden 狀態後，飼主端清單應同步顯示為灰色「隱藏中」狀態。
- [ ] **AC-4 (隱私驗證)**：確認加入收藏後，保母端不會收到任何通知 (PRD-014 訊息中心無紀錄)。
- [ ] **AC-5 (移除收藏)**：執行移除後，清單應正確消失該項目。

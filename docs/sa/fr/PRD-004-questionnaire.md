## 背景與目的

為了降低溝通成本並精準評估接單風險，保母通常需要在預約階段收集特定的毛孩資訊 (如：是否會攻擊人、上次打疫苗時間等)。本功能允許保母自主設計「事前問卷」，透過動態表單技術，讓每一位保母都能根據自己的專業領域客製化提問流程。

## 使用者故事

- 身為 **保母**，我希望能新增自定義問題，並選擇不同的回答格式 (如單選、多選、簡答)，以便收集我需要的資訊。
- 身為 **保母**，我希望能設定問題是否為「必填」，確保飼主提供關鍵訊息。
- 身為 **保母**，我希望能透過拖曳調整問題順序，讓問卷內容呈現更具邏輯性。
- 身為 **保母**，我希望能隨時啟用或停用某個問題，而不必刪除它，以保留歷史紀錄。

## 功能規格

### 主流程

**A. 問卷問題 CRUD 管理**
1. 保母進入「服務設定 > 事前問卷」頁面。
2. 點擊「新增問題」，設定以下內容：
   - **題目內容**：具體的提問文字。
   - **回答類型**：單選 (Radio)、多選 (Checkbox)、簡答 (Input)、詳答 (Textarea)。
   - **選項內容**：若為單選/多選，需輸入選項清單。
   - **必填設定**：開關 (Yes/No)。
3. 系統驗證內容後儲存，並賦予預設排序值。

**B. 問題排序與狀態控制**
1. 在問卷管理清單中，支援列表拖曳調整排序順位。
2. 每題設有「啟用/停用」開關。
3. 停用後的問題將不再出現在飼主的預約流程中，但歷史訂單的答案仍會保留。

### 例外處理

| 情境 | 預期行為 |
|------|---------|
| 嘗試修改已被飼主回填過的問題類型 (如單選改簡答) | 系統允許修改，但需提示「修改問題類型可能影響已填寫的舊答案顯示，建議停用舊題並新增新題」。 |
| 刪除已有回傳答案的問題 | 系統採邏輯刪除，確保歷史訂單中的回覆資料不會遺失。 |
| 必填問題未設定任何選項 (針對單/多選) | 系統阻擋儲存並提示「單選或多選題必須至少設定一個選項」。 |

### 資料規則 / 業務規則

- **動態性**：問卷內容不固定，需支援動態增減，並在 `PRD-005` 預約頁面依據此處設定渲染表單。
- **排序邏輯**：採遞增排序，數字越小排越前面。
- **回答完整性**：若問題設為必填，系統在預約提交時必須強制驗證該題是否有值。
- **歷史隔離**：問題的異動不應直接修改已提交訂單的答案快照紀錄，以維持合約一致性。

## 驗收標準（AC）

- [ ] **AC-1 (問題建立)**：保母可成功建立不同類型的問題 (單選/多選/簡答)，且選項能正確儲存。
- [ ] **AC-2 (排序持久化)**：手動調整問題順序並儲存後，重新整理頁面順序應維持不變。
- [ ] **AC-3 (狀態檢核)**：被停用的問題絕對不能出現在 PRD-005 的飼主預約頁面。
- [ ] **AC-4 (必填驗證)**：若某題設為必填，在預約流程中未填寫應無法進入下一步。
- [ ] **AC-5 (編輯/刪除保護)**：對已有答案的問題進行編輯時，系統有給予明確的影響提示。

## 備註 / 設計限制

- **題目長度限制**：題目文字上限 200 字，選項上限 50 字。
- **題數限制**：單一保母建議上限 20 題，以維持使用者體驗。

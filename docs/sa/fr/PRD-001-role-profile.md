## 背景與目的

本系統為雙邊平台 (Dual-sided Marketplace)，使用者可同時兼具「保母」與「飼主」身分。為了降低註冊門檻並提升體驗，系統需支援單一帳號無縫切換雙角色，且資料需嚴格依據 `profile_id` 隔離。
同時，為滿足進階保母管理客戶主權的需求，引入「預約門禁 (Gatekeeper)」機制，允許專業版以上保母針對「特定方案」或「全域範圍」，透過客戶的唯一 ID 進行黑/白名單控管。

## 使用者故事

- 身為 **使用者**，我希望能一鍵切換「保母」與「飼主」身分，以便在同一個 APP 內處理接單與送單需求。
- 身為 **專業版 (Pro) 保母**，我希望能將特定客戶列入黑名單，使其無法看到我的預約網頁或特定方案。
- 身為 **頂級版 (Premium) 保母**，我希望能設定特定方案僅開放給白名單客戶 (熟客) 預約，實現精準經營。

## 功能規格

### 主流程

**A. 角色切換流程**
1. 使用者於前端介面點擊「切換角色 (Sitter/Client)」按鈕。
2. 前端呼叫角色切換 API (或純前端 Context 切換，若 JWT 已包含雙 Profile ID)。
3. 前端清空當前視圖狀態，切換全局主題色 (保母：Amber / 飼主：Blue)。
4. 重新導向至目標角色的預設首頁 (Dashboard)。

**B. 預約門禁設定流程 (限保母端 - 我的 Profile)**
1. 保母進入「預約門禁設定」頁面。
2. 系統檢核當前保母的訂閱方案權限：
   - 基礎版 (Basic)/免費版 (Free)：顯示功能鎖定與升級提示。
   - 專業版 (Pro)：開放「黑名單」模式設定。
   - 頂級版 (Premium)：開放「黑名單」與「白名單」模式設定。
3. 保母選擇門禁的「套用範圍」(全域或指定特定服務方案)，並輸入欲管控的目標帳號唯一 ID 列表。
4. 系統儲存「預約門禁規則」設定。

### 例外處理 (Cemetery Path)

| 情境 | 預期行為 |
|------|---------|
| JWT 憑證逾期時點擊切換角色 | 攔截操作，清除 token，將使用者導向 Login 頁面 (401 Unauthorized)。 |
| 權限不足的保母嘗試呼叫設定 API | 後端回傳 `403 Forbidden`，前端提示「此為專業版/頂級版功能，請升級方案」。 |
| 保母輸入無效的帳號/Email格式 | 前端擋下並提示，後端回傳 `400 Bad Request` (需符合 Email 格式規範等)。 |
| 同時將同一號碼加入黑名單與白名單 | 系統阻擋並提示衝突，預設黑名單優先級高於白名單。 |

### 資料規則 / 業務規則

- **權限隔離與請求防護**：系統在處理任何業務資源 (如：訂單、行程) 時，必須由伺服器端自動識別當前的「操作身分」與「角色」，嚴禁由前端（客戶端）自行指定身分發送請求，以防越權操作。
- **接單資格卡控 (KYC Gating)**：保母的公開預約網頁僅在其檔案狀態為 `VERIFIED` 時才能正常顯示。詳細審核流程見 PRD-017。
- **預約門禁邏輯 (Gatekeeper Logic)**：
  - **全域封鎖**：若帳號在全域黑名單中，進入預約網頁時將直接顯示「保母目前不開放預約」。
  - **方案隱藏制**：若帳號在特定方案的黑名單中，或不在該方案的白名單中，預約時該方案將自動從清單中隱藏。
  - **免填問卷清單**：保母可將特定「老客戶」或「信任客戶」加入此清單。被加入者在預約流程中將自動跳過問卷步驟。
- **內部信用指標 (Internal Trust Score)**：
  - **屬性定義**：保母角色資料中包含一項「內部信用指標」，初始狀態定義為 100 點。
  - **隱私級別**：**極高度隱私**。此資訊僅供 `PRD-020` 管理後台調閱與操作，保母與飼主端的所有介面皆不得顯示此項指標。
- **降級處理 (Degradation)**：若保母的頂級版方案逾期未繳費自動降為免費版，系統不刪除白名單資料，但該門禁設定自動失效（視為 OPEN），直到重新付費。

## 驗收標準（AC）

- [ ] **AC-1 (角色切換UI)**：切換角色後，全局的主題色與底部導覽列 (Bottom Nav) 必須正確變更。
- [ ] **AC-2 (狀態隔離)**：切換至飼主端後，無法透過前端路由強行存取保母端的頁面 (例如 `/sitter/finances`)。
- [ ] **AC-3 (權限檢核)**：Free 與 Basic 方案的保母進入門禁設定頁面，只能看到功能介紹與升級按鈕。
- [ ] **AC-4 (黑名單設定)**：Pro 方案保母可成功新增、刪除帳號至黑名單，且不能選用白名單模式。
- [ ] **AC-5 (白/黑名單設定)**：Premium 方案保母可自由切換門禁模式，且系統能正確儲存名單設定。
- [ ] **AC-6 (資料格式)**：帳號或 Email 必須通過 Regex 驗證。
- [ ] **AC-7 (方案降級保護)**：保母方案降級時，原先設定的門禁規則將被標記為失效/隱藏，不影響系統正常預約。

## 備註 / 設計限制

- **架構考量**：若使用者註冊時預設只建立了一種 Profile，角色切換流程需加入「動態建立 Profile」的步驟（Lazy Initialization）。
- **隱私問題**：白名單與黑名單中的帳號或 Email 應加密儲存或遮蔽，並在 SA/SD 階段確認是否符合個資法要求。

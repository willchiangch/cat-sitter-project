## 背景與目的

服務執行中產生的照片與影片紀錄是飼主最關心的內容。然而，隨著服務量增加，多媒體檔案將佔用大量系統資源。本模組旨在建立一套符合業務邏輯的保留機制，根據保母的訂閱等級決定檔案的存活時間，在使用者體驗與維運成本之間取得平衡。

## 使用者故事

- 身為 **保母**，我希望能清楚各方案的照片保留天數，以便我在服務開始前能正確告知飼主。
- 身為 **飼主**，當服務報告中的照片即將到期時，我希望能收到系統提醒，以免遺失毛孩的珍貴影像。
- 身為 **系統管理員**，我希望系統能自動清理超時的檔案，不需要人工介入。

---

## 功能規格

### 1. 檔案保留期限規則

系統依據「預約成立時」保母所屬的訂閱等級，鎖定該筆訂單所有多媒體紀錄的保留天數規格。該規格一旦鎖定，不隨保母後續方案到期或降級而追溯縮短。

| 方案等級 | 檔案保留期限 | 起算基準點 |
|:--- |:--- |:--- |
| **體驗版 (Free)** | 7 天 | 訂單結案日 |
| **新手版 (Basic)** | 30 天 | 訂單結案日 |
| **專業版 (Pro)** | 90 天 | 訂單結案日 |
| **頂級版 (Premium)** | **永久保留** | 帳號存續期間 |

### 2. 業務行為與流程

**A. 效期鎖定與動態延長機制**
- **合約規格快照**：系統於訂單確認 (`CONFIRMED`) 的當下，記錄當時方案對應之保留天數。
- **降級契約保護**：若保母在服務期間發生「方案降級」，該筆已成立之訂單仍依據成交時的優渥效期進行保留。
- **升級追溯延長**：若保母於檔案尚未過期前「升級方案」，系統應提供追溯展延機制，將尚未刪除之檔案依據新方案規則重新計算並延長移除日期。

**B. 逾期預告通知**
- 飼主查看報告頁面時，系統應顯示明顯的「存檔提醒」，標註照片將於何時移除。
- **自動提醒**：在移除日期前 3 天，系統應主動通知飼主，提醒備份照片。

**C. 自動清理行為**
- 系統應在檔案到達移除日期的翌日執行自動移除作業。
- **狀態轉換**：移除後的檔案在報告頁面上應呈現「檔案已逾期移除」之文字提示，原圖/原影片連結失效。

### 3. 例外與限制規則

- **未結案訂單**：若訂單遲未結案 (狀態非 DONE)，則檔案視為「服務中狀態」，不啟動移除計時。
- **永久保留例外**：頂級版方案之檔案僅在使用者主動註銷帳號時，才隨帳號資料一併移除。
- **不可逆原則**：檔案一旦過期移除，系統不提供任何形式的還原或垃圾桶機制，此條款需列入保母付費條款。

## 驗收標準（AC）

- [ ] **AC-1 (規則正確性)**：體驗版保母的訂單結案後，系統計算的移除日期必須精準為 7 天後。
- [ ] **AC-2 (提醒顯示)**：飼主在手機端查看報告時，必須能看到移除日期的倒數或具體日期。
- [ ] **AC-3 (自動移除驗證)**：模擬日期到達移除日後，檔案連結應無法再開啟，且頁面顯示逾期提示。
- [ ] **AC-4 (升級展延驗證)**：體驗版保母升級為專業版後，其尚未被刪除之舊照片，移除日期應自動變更為結案日 + 90 天。
- [ ] **AC-5 (降級隔離驗證)**：保母方案降級後，降級前產生的照片移除日期不可被縮短，仍依原合約保留。

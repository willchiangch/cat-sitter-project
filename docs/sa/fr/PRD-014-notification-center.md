## 背景與目的

為了確保使用者能即時掌握訂單動態、服務進度與重要帳號資訊，系統需提供一個統一的「訊息中心」。透過紅點提示與訊息清單，引導使用者完成必要操作，並減少因遺漏訊息而導致的服務延遲或爭議。

## 使用者故事

- 身為 **使用者 (保母/飼主)**，我希望能有一個專屬的清單，統一查看所有系統發給我的訊息。
- 身為 **使用者**，我希望能透過紅點快速感知是否有新訊息進來。
- 身為 **使用者**，我希望能點擊訊息後直接跳轉到該訊息關聯的功能頁面，省去手動尋找的時間。
- 身為 **使用者**，我希望能針對不同類別的訊息設定是否接收，避免被打擾。

---

## 功能規格

### 1. 業務行為與流程

**A. 訊息提示與狀態管理**
- 系統介面需提供「訊息通知入口 (小鈴鐺)」，並顯示「未讀訊息總數」。
- 點擊入口展開訊息清單，內容應包含：訊息標題、簡短內容、發送時間。
- **讀取狀態**：點擊單則訊息後，該訊息標示為已讀，未讀計數隨之減少。
- **一鍵處理**：提供「全部標示為已讀」功能，快速清除所有紅點提示。

**B. 訊息自動引導 (跳轉行為)**
- 系統發送訊息時需關聯特定的「功能頁面」。
- 使用者在訊息清單點擊該則訊息時，系統需具備自動導向功能。
  - 例如：收到「有新預約」訊息，點擊後應直接顯示該筆預約的內容。
  - 例如：收到「KYC 被退件」訊息，點擊後應直接進入認證上傳頁面。

**C. 通知類別與偏好控管**
- 使用者可在個人設定中，針對不同類別的通知選擇接收方式 (如：站內信或 Email)。
- **安全控管鎖定**：涉及帳號安全、合約異動或付費續訂等關鍵訊息，系統應強制開啟，不允許使用者關閉。

### 2. 系統通知類別定義

| 類別 (Category) | 描述 | 重要性 |
|:--- |:--- |:--- |
| **訂單事務** | 包含新預約、異動通知、結案提醒、服務報告送達等。 | 高 |
| **帳號認證** | 實名認證 (KYC) 的進度與審核結果。 | 極高 |
| **訂閱與維護** | 方案到期提醒、自動降級告知、系統維護公告。 | 高 |
| **服務紀錄** | 照護過程的照片/影片即將移除之提醒。 | 中 |

### 3. 業務規則與限制

- **歷史紀錄保存**：為確保系統反應速度，訊息中心僅保留最近 90 天內的紀錄。
- **認證優先權**：在保母尚未完成認證前，系統應優先推送 KYC 相關訊息，直到認證完成。
- **權限連動**：訊息內容應受到角色隔離。保母不應收到飼主的系統通知，反之亦然。

## 驗收標準（AC）

- [ ] **AC-1 (提示精準度)**：當系統發送新訊息時，小鈴鐺的未讀數字應立即更新。
- [ ] **AC-2 (跳轉有效性)**：測試點擊不同類別的訊息 (如訂單、認證)，是否能正確抵達對應的功能頁面。
- [ ] **AC-3 (已讀同步)**：點擊單則訊息或執行「全部標示已讀」後，未讀計數必須與列表狀態保持同步。
- [ ] **AC-4 (偏好鎖定驗證)**：在設定頁面中，確認「關鍵類別」的開關無法被手動關閉。
- [ ] **AC-5 (自動清理驗證)**：超過 90 天的舊訊息不應出現在使用者的訊息清單中。

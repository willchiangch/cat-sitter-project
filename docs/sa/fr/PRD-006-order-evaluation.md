## 背景與目的

當飼主送出預約申請後，訂單進入「待評估」狀態。保母需依據飼主提供的毛孩資訊、事前問卷回覆以及自身的檔期狀況進行審核。此功能賦予保母「最終決定權」與「報價調整權」，確保在正式收款前，雙方對於服務內容與價格已達成一致共識。

## 使用者故事

- 身為 **保母**，我希望能一目了然地看到預約申請的所有細節 (日期、毛孩特徵、問卷回覆)，以便判斷是否接單。
- 身為 **專業版 (Pro) 以上保母**，我希望能針對特殊需求 (如：車馬費、特殊用藥、多毛加價) 進行手動金額調整。
- 身為 **保母**，我希望能送出最終報價給飼主，並通知其進行付款，以啟動正式服務。
- 身為 **保母**，我希望能禮貌地拒絕不合適的預約請求。

## 功能規格

### 主流程

**A. 訂單審核與問卷決策**
1. 保母收到通知後進入「訂單詳情」頁面。
2. 系統自動識別並提示飼主狀態：
   - **白名單熟客**：系統提示「此為免填問卷熟客」，保母可直接進入 B 段進行契約制定。
   - **新客戶 / 非白名單**：
     - 若保母認為資料已足夠：可點擊「加入免填清單」，並直接進入 B 段。
     - 若需更多資訊：點擊「**要求填寫問卷**」，系統發送連結給飼主。訂單狀態維持在待評估，直到飼主完成。
3. 頁面彙整：行程資訊、毛孩基礎資料、以及（若已填寫的）問卷回覆。

**B. 價格手動調整 (限專業版以上)**
1. 保母點擊「調整價格」按鈕。
2. 系統顯示「原始試算總額」。
3. 保母輸入「調整金額」(加價或減價) 與調整原因。
4. 系統自動計算並顯示「最終報價總額」。

**C. 送出報價 (Confirm & Quote)**
1. 保母確認「報價金額」無誤後點擊「送出報價」。
2. 系統動作：
   - 訂單狀態更新：`PENDING` -> `PENDING_PAYMENT`。
   - 鎖定訂單金額（行程時間、方案種類、數量、最終報價）。
   - 發送「報價已送出，請確認並付款」通知給飼主。

**D. 拒絕預約 (Reject)**
1. 保母點擊「婉拒預約」，可選擇/填寫簡短原因。
2. 系統動作：
   - 訂單狀態更新：`PENDING` -> `CANCELLED`。
   - 記錄日誌 `ORDER_REJECTED`。
   - 發送「預約已婉拒」通知給飼主 (含 Email 與站內訊息)。

### 例外處理

| 情境 | 預期行為 |
|------|---------|
| 基礎版保母嘗試調整價格 | 系統顯示功能鎖定，提示「手動調價為專業版功能」，並引導升級。 |
| 飼主在保母審核中嘗試取消 | 允許取消，訂單狀態轉為 `CANCELLED`，保母端頁面顯示「飼主已取消預約」。 |
| 報價金額調整後總額為負數 | 系統阻擋並提示「訂單總額不得為負數」。 |

### 資料規則 / 業務規則

- **報價有效期**：(未來擴充) 可設定報價後 24 小時未付則自動失效。
- **權限限制**：
  - **免費版/基礎版**：僅能執行「接受(原價)」或「拒絕」，無法設定調整金額。
  - **專業版/頂級版**：開放「調整金額」功能。
- **金額計算公式**：`最終報價金額 = 原始試算金額 + 調整金額`。
- **金額精準度**：所有金額欄位必須採 `BigDecimal(18,4)` 處理。
- **不可逆性**：一旦送出報價進入 `PENDING_PAYMENT`，訂單金額、行程時間與方案內容皆不可再被編輯。
- **訂單內容範圍**：訂單僅載明「行程時間、方案種類、數量、調整金額與最終金額」，不包含執行工項清單（工項記錄詳見 PRD-021）。

## 驗收標準（AC）

- [ ] **AC-1 (資訊彙整)**：保母在審核頁面能正確看到該筆訂單關聯的行程、方案、毛孩基礎資料與問卷答案。
- [ ] **AC-2 (專業版調價)**：專業版帳號能成功輸入調整金額 (增/減) 與原因，且訂單能正確呈現「原始金額、調整金額、最終總額」三項數據。
- [ ] **AC-3 (狀態移轉-接受)**：執行送出報價後，訂單狀態必須轉為 `PENDING_PAYMENT`。
- [ ] **AC-4 (狀態移轉-拒絕)**：執行婉拒預約後，訂單狀態必須轉為 `CANCELLED`。
- [ ] **AC-5 (編輯鎖定)**：進入 `PENDING_PAYMENT` 狀態的訂單，金額與行程資訊欄位必須鎖定，不可編輯。
- [ ] **AC-6 (無工項契約)**：訂單詳情頁面不得出現任何執行工項清單或 SOP 相關欄位。

## 備註 / 設計限制

- **修改限制**：本階段保母僅能調整「價格」，若需修改「日期」，建議請飼主取消並重新發起預約。

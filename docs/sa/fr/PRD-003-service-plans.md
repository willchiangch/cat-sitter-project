## 背景與目的

服務方案是保母接單的核心，定義了照護內容、收費標準與開放時間。為了應對非標化的照護需求，系統需提供靈活的方案設定功能，並透過訂閱機制區分「常態開放」與「進階檔期管理」功能，協助專業保母提升經營效率與收益。

## 使用者故事

- 身為 **保母**，我希望能建立多種服務方案 (如：到府餵食、陪玩、過夜照護)，並設定不同的單價與詳細描述，以便飼主選擇。
- 身為 **保母**，我希望能對方案進行排序與上/下架管理，以便靈活調整我的服務重點。
- 身為 **專業版 (Pro) 以上保母**，我希望能針對特定方案設定「可預約日期區間」(如：特定假日限定方案)，避免在非服務時段收到預約請求。

## 功能規格

### 主流程

**A. 服務方案 CRUD 管理**
1. 保母進入「服務設定 > 方案管理」頁面。
2. 填寫方案詳細資訊：
| 項目 | 說明 | 格式 |
|------|------|------|
| **方案名稱** | 如：30 分鐘到府照護、全日居家待命。 | 必填 |
| **基礎單價** | 該方案的起標價格。 | 必填 (BigDecimal) |
| **預設任務清單 (SOP)** | 該方案預設包含的照護項目 (如：餵食、更換砂盆)。 | **建議模板，可於訂單評估時修改** |
| **開放預約日期** | 保母可設定特定的日期區間才開放此方案。 | 必填 |
   - **適用寵物種類**：提供「狗/貓/鳥/鼠/兔/爬蟲/昆蟲/其他」之多選清單。
   - **詳細描述**。
   - **是否生效**：可選擇「常態生效」或設定「生效區間 (例如：僅限暑假期間自動開放預訂，時間過後自動關閉下架)」。
   - **預約門禁 (白名單)**：可綁定 `PRD-001` 設定之白名單，限定只有名單內之客戶可見此方案。
3. 系統驗證必填項與價格格式後，儲存方案並預設為「已上架」。
4. 支援列表拖曳排序功能。

**B. 檔期控制 (可預訂日期區間 - 限專業版以上)**
1. 保母編輯方案時，可開啟「限定日期開放預訂」功能。
2. 設定 `可預訂日期區間` (例如：跨年連假方案僅開放 12/30~1/2 提供服務)。
3. 系統將在飼主端預約網頁自動檢核：若預約目標日期不在該方案設定的區間內，該方案顯示為不可選。

### 例外處理

| 情境 | 預期行為 |
|------|---------|
| 嘗試下架已有進行中訂單的方案 | 系統允許下架，但僅影響「未來」的預約。已成立的訂單仍依據成交當時的方案內容執行。 |
| 基礎版保母嘗試設定日期區間 | 前端顯示功能鎖定，引導保母點擊升級方案。 |
| 價格輸入負數或非數字 | 系統阻擋並提示「價格必須為大於 0 的有效數字」。 |

### 資料規則 / 業務規則

- **價格精度**：方案單價必須以 `BigDecimal(18,4)` 處理，避免浮點運算誤差。
- **權限限制 (SaaS Tiers)**：
  - **免費版/基礎版**：方案僅能設定為「常態開放」，無法限制日期。
  - **專業版/頂級版**：開放手動設定方案的 `開放預約起訖日`。
- **下架邏輯**：下架後的方案將從飼主預約網頁消失，但資料庫端需保留，以利歷史訂單回溯。

## 驗收標準（AC）

- [ ] **AC-1 (方案建立)**：保母可成功建立含有名稱、單價、時長、寵物種類與描述的服務方案。
- [ ] **AC-2 (生效區間切換)**：當系統時間到達方案設定的「生效區間」終點時，方案應自動從前台預約網頁下架隱藏。
- [ ] **AC-3 (專業版檔期卡控)**：飼主預約的日期若不在方案設定的「可預訂日期區間」內，該方案必須呈現 Disabled 狀態。
- [ ] **AC-4 (門禁白名單)**：設定白名單的方案，若非白名單內的飼主登入，預約頁面上應完全隱藏此方案。
- [ ] **AC-5 (價格格式驗證)**：系統必須攔截無效的價格輸入，並給予清晰的紅字提示。

## 背景與目的

服務執行是保母展現專業價值的核心環節。保母到府後需依據標準作業程序 (SOP) 進行照護，並透過照片、影片即時紀錄毛孩狀況。系統需提供流暢的執行面板，並根據保母的訂閱方案動態調整多媒體紀錄的上限，最後匯整產生照護報告，讓飼主感到安心。

## 使用者故事

- 身為 **保母**，我希望能依照當日的行程清單開始服務 (Check-in)，並在完成每一項照護任務時進行勾選。
- 身為 **保母**，我希望能快速拍攝照片或錄製短影片上傳，以便與飼主分享毛孩現狀。
- 身為 **保母**，當我嘗試上傳超過方案限制數量的照片時，系統能提醒我並建議升級方案。
- 身為 **保母**，我希望能一鍵產生並送出今日的服務報告給飼主。

## 功能規格

### 主流程

**A. 今日行程列表顯示與排序**
1. 保母進入系統查看當日的服務行程清單。
2. 系統依據以下邏輯進行排序：
   - **已排定時間之行程**：優先顯示，依據「預計執行時間」先後順序排列。
   - **未排定時間之行程 (全天事件)**：列於列表下方，依據訂單建立時間排序。

**B. 執行服務與紀錄 (SOP)**
1. 保母到達現場後，直接於列表中選擇對應行程進入「執行面板」。
2. 系統展示該筆訂單在評估階段（PRD-006）所制定的「**執行契約任務清單**」。
3. 保母每完成一項，進行勾選並可輸入「單項備註」。

**C. 多媒體日誌與報告提交**
1. 保母拍攝照片或錄製短影片上傳至當前行程，系統依據方案等級限制數量。
2. 保母確認任務清單、備註與多媒體內容無誤後，填寫「總結心得」並點擊「送出報告」。
3. 系統動作：
   - 記錄報告送出之時間與位置資訊（內部稽核用）。
   - 行程狀態更新：直接由 `PENDING` 轉為 `DONE`。
   - **發送服務報告**：系統即時發送該次行程的完整報告給飼主。
   - **發送頻率原則**：報告以「單次行程」為發送單位。若同一天有多個行程，則每完成一次即發送一次，不進行跨行程彙整，以確保即時性。

**D. 服務報告內容組成**
一份完整的服務報告應包含以下資訊：
- **基礎資訊**：執行保母、行程日期、實際完工時間。
- **任務清單紀錄**：各項 SOP 任務的勾選狀態與其對應的單項備註。
- **多媒體紀錄**：本次服務上傳的所有照片與影片。
- **總結心得**：保母針對該次服務與毛孩狀態的整體描述。
- **異常註記**：若有特殊突發狀況（如嘔吐、受傷）之特別提醒。

### 例外處理

| 情境 | 預期行為 |
|------|---------|
| 網路收訊不良處上傳日誌 | 應用程式端執行「背景非同步傳輸」與「失敗自動重試」機制，不卡住保母操作流程。 |
| 預計服務日結束後仍未提交報告 | 系統在翌日清晨自動將未提交報告的行程標記為「未執行」，並通知保母處理。 |
| 地理位置偏差過大 | 系統在提交報告時記錄位置偏差，不阻擋操作，但於內部管理介面標註警示。 |

### 資料規則 / 業務規則

- **媒體規格與壓縮**：
  - 照片：前端自動壓縮至解析度 1920px 以下，大小 ≤ 1MB。
  - 影片：解析度上限為 **720p**，單一檔案上限為 **50MB**。
  - **前端責任**：錄製或上傳時，應優先由客戶端進行轉碼或限制錄製規格。
- **不可修改性**：一旦點擊「送出報告」，該行程的內容（含照片、備註）轉為唯讀，不可再修改。
- **地理位置紀錄**：提交報告時需記錄位置資訊以供內部稽核。

## 驗收標準（AC）

- [ ] **AC-1 (服務啟動)**：保母點擊開始後，行程狀態能即時更新，並正確記錄系統時間。
- [ ] **AC-2 (媒體卡控)**：不同方案等級的帳號，在上傳時必須嚴格遵守對應的數量與類型限制。
- [ ] **AC-3 (狀態移轉-完工)**：送出報告後，該行程在清單中應標記為「已完成」，且內容變為唯讀。
- [ ] **AC-4 (背景上傳驗證)**：在模擬網路斷線後恢復的情況下，上傳任務應能自動完成。

## 背景與目的

專業貓咪保母的檔期通常非常緊湊，當遇到熱門連假或個人因素無法接單時，為了不讓長期信任的飼主失望，保母通常會希望能將案件轉介紹給自己信任的同行。本功能旨在建立保母間的「信任圈」，透過系統化的轉介機制，一方面解決保母滿單的問題，另一方面也為飼主提供高品質的替代選擇。

## 使用者故事

- 身為 **保母**，我希望能將合作愉快且專業能力受認可的同行加入「信任圈」，以便未來互相轉單。
- 身為 **保母**，即便飼主尚未正式下單，我也能在溝通後主動分享信任圈內的保母卡片給飼主。
- 身為 **保母**，當我因檔期全滿無法接單時，我希望能一鍵將該預約轉介給信任圈內的保母，並附上推薦原因。
- 身為 **飼主**，當我心儀的保母無法服務時，我希望能收到他推薦的保母名單，增加找到合適人選的效率。

## 功能規格

### 主流程

**A. 信任圈管理 (保母間)**
1. 保母進入「設定 > 信任圈」頁面。
2. 透過搜尋功能 (依帳號或 Email) 找到特定保母，發送「信任請求」。
3. 對方收到通知並同意後，雙方正式建立信任關係，出現在彼此的信任圈清單中。
4. 任何一方可隨時移除信任關係。

**B. 轉介發起流程**
1. 保母在審核訂單 (PRD-006) 或收到預約諮詢時，若決定不接單，可選擇「不接單並轉介」。
2. 系統彈出信任圈清單。
   - **黑名單前置過濾**：系統自動排除「已將該飼主列入黑名單」的信任圈保母，其不會顯示於可選清單中，避免飼主漸推薦連結後遭遇封鎖，導致体驗極差。
3. 保母選擇 1 位或多位推薦對象。
4. 保母可填寫一段推薦語（例如：這位保母也很細心，特別擅長照顧老貓）。

**C. 轉介通知與互動**
1. **對飼主**：系統發送「保母推薦」通知。飼主可點擊連結查閱推薦保母的公開檔案與預約連結。
2. **對被推薦保母**：系統發送「收到轉介」通知。告知其由哪位保母轉介、哪位飼主可能有需求。

**D. 主動分享轉介 (非訂單關聯)**
1. 保母進入「信任圈」清單。
2. 針對特定信任對象點擊「推薦此保母」。
3. 系統產生「推薦分享連結」(包含推薦語與推薦對象的 Profile 連結)。
4. 保母可：
   - **複製連結**：透過第三方通訊軟體 (如 Line) 傳給飼主。
   - **發送至站內帳號**：可透過輸入飼主的 **Email** 或 **系統帳號 ID** 進行搜尋，直接發送站內轉介通知。

### 例外處理

| 情境 | 預期行為 |
|------|---------|
| 被推薦的保母目前也處於停權或休假狀態 | 轉介清單中應標註狀態，或暫時隱藏該對象。 |
| 飼主拒絕接受轉介建議 | 流程結束，系統不進行任何後續動作。 |
| 嘗試轉介給非信任圈內的保母 | 系統不允許此操作，必須先建立信任關係。 |

### 資料規則 / 業務規則

- **隱私保護**：轉介過程僅提供推薦保母的「公開 Profile 連結」，系統不應主動將飼主的私密聯絡資訊直接提供給被推薦保母，直到飼主主動聯繫對方。
- **轉介性質**：本功能僅為「資訊媒介」，轉介後的後續預約、付款與服務品質，由飼主與新的保母重新建立合約關係，原轉介保母不負連帶責任。
- **雙向同意制**：信任圈必須經由雙方確認才成立，防止單方面掛名推薦。

## 驗收標準（AC）

- [ ] **AC-1 (建立信任關係)**：保母 A 發送請求，保母 B 同意後，雙方的信任圈清單皆能正確顯示對方。
- [ ] **AC-2 (轉介操作)**：在婉拒訂單的流程中，保母能順利選取信任圈內的對象並送出轉介。
- [ ] **AC-3 (通知送達)**：飼主與被推薦保母皆應在 1 分鐘內收到系統通知。
- [ ] **AC-4 (推薦語呈現)**：飼主收到的通知內容中，應能完整看到原保母填寫的推薦語。
- [ ] **AC-5 (解約權限)**：移除信任關係後，原本的轉介清單中應立即消失該對象。

## 備註 / 設計限制

- **商業分潤**：本階段不考慮轉介佣金或分潤機制，純粹為互助性質。

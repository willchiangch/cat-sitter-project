## 1. 身分認證安全 (Authentication)

**思維：身分驗證是系統的第一道防線，必須在便利與安全間取得平衡。**

*   **密碼安全保護**：所有使用者密碼在系統中必須經過「單向不可逆雜湊技術」處理，嚴禁以任何形式紀錄明文密碼，確保即便資料外流也無法還原原始密碼。
*   **存取憑證管理**：
    *   使用者登入後核發之「身分權杖 (Token)」有效期限不得超過 24 小時。
    *   系統需支援「憑證更新機制 (Refresh Mechanism)」，在不影響安全性的前提下維持使用者的登入狀態。
*   **暴力破解防護**：單一帳號若連續登入失敗達 5 次，系統應暫時鎖定該帳號之存取功能 10 分鐘，並記錄該次異常嘗試。

## 2. 資料與多媒體保護 (Data Protection)

**思維：個資與影像資料是平台最敏感的資產，必須落實分級儲存與加密。**

*   **敏感個資加密**：
    *   涉及個人身分之欄位（如身分證號、銀行帳號）在系統儲存層必須進行「軍規等級的對稱式加密 (AES-256)」。
    *   金鑰管理應與業務資料隔離存放。
*   **多媒體檔案隱私 (KYC 專屬)**：
    *   保母上傳的身分證件照、手持證件自拍照必須存放在「不公開存取」的私有空間。
    *   除了具備審核權限的管理員外，任何外部網址直接存取皆應被拒絕。
*   **傳輸安全**：全站所有對外服務一律強制使用 HTTPS 加密傳輸協定，禁止任何非加密的通訊路徑。

## 3. 權限控管與防禦 (Authorization & Defense)

**思維：落實「最小權限原則」，確保每個人只能看到他該看的。**

*   **角色存取過濾**：
    *   所有服務請求必須經過後端嚴格校驗：確保「飼主」無法存取「保母」的私有資訊（如：接單統計），反之亦然。
    *   管理員功能必須具備獨立的存取權限控管。
*   **關鍵操作二次驗證**：
    - **強制觸發場景**：修改密碼、變更銀行撥款帳戶。
    - **驗證方式**：MVP 階段優先採用「輸入原登入密碼」進行二次確認。
    - **生物辨識 (WebAuthn)**：支援 FaceID / TouchID / 指紋辨識登入，因系統註冊時已執行強驗證，執行業務操作時無需額外 2FA。
*   **全域存取限流 (Rate Limiting)**：
    - **非登入使用者**：單一 IP 每分鐘最多 60 次 API 請求，防範保母公開檔案被爬蟲大量擷取。
    - **登入使用者**：根據業務合理性設定較寬鬆上限。
*   **輸入安全檢核**：系統對所有使用者輸入之內容應進行自動化過濾，確保網頁呈現之安全性。

---

## 4. 安全稽核路徑 (Audit Trail)

*   **關鍵異動紀錄**：針對使用者身分狀態變更、權限異動、個資修改等行為，系統必須留下不可抹除的操作紀錄，包含操作者、時間、以及異動項目。

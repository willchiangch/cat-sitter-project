# 非功能性需求 (Non-Functional Requirements) 大綱

> **建立日期**：2026-05-09
> **狀態**：Draft
> **適用範圍**：貓咪保母 PWA 全系統

---

## NFR-001: 效能需求 (Performance)

### 回應時間
- 一般頁面載入：首次載入 ≤ 3 秒、後續換頁 ≤ 1.5 秒。
- 訂單送出 / 報價確認等寫入操作：回應時間 ≤ 2 秒。

### 多媒體處理
- 照片上傳：前端自動壓縮至 1MB 以下再傳送，單張上傳完成 ≤ 5 秒 (4G 網路環境)。
- 影片上傳：支援背景非同步上傳，不阻塞使用者操作。

### 行事曆同步
- 訂單狀態變更後，行事曆事件更新延遲應 ≤ 5 分鐘。

---

## NFR-002: 可用性與可靠性 (Availability & Reliability)

### 系統可用性
- 目標 SLA：99.5% (每月停機時間 ≤ 3.6 小時)。
- 排程維護作業 (如凌晨清理排程) 不得影響前端使用者操作。

### 錯誤容忍
- 第三方服務 (Google Calendar、金流 API) 無法連線時，系統核心功能 (接單/報價/服務紀錄) 不得中斷。
- 金流回呼 (Webhook) 失敗時，需具備重試機制，確保交易最終一致性。

---

## NFR-003: 安全性 (Security)

### 身份認證
- 密碼儲存一律採用 bcrypt 或 Argon2 雜湊，禁止明文。
- JWT Token 應設定合理的有效期限 (建議 ≤ 24 小時)，並支援 Refresh Token 機制。
- 連續登入失敗 5 次需鎖定帳號 10 分鐘。

### 資料保護
- KYC 身分證照片必須儲存於私有權限的儲存空間，僅限後台管理員存取。
- 所有 API 傳輸一律強制使用 HTTPS。
- 個資欄位 (如身分證號、銀行帳號) 需在系統內進行加密處理 (AES-256 或同等強度)。

### 權限控管
- 所有 API 必須驗證操作者身分與角色 (保母/飼主/管理員)，防止越權存取。
- 敏感操作 (修改密碼、綁定生物辨識、銀行帳戶變更) 需進行二次驗證。

---

## NFR-004: 可擴展性 (Scalability)

### 使用者規模
- MVP 階段目標承載：500 位活躍保母、5,000 位活躍飼主。
- 系統架構設計需能水平擴展，預留年增長 3 倍之空間。

### 儲存規模
- 多媒體檔案預估量：每月新增 50GB (依活躍保母數與方案分布)。
- 系統需具備根據 PRD-013 保留策略自動回收儲存空間之能力。

---

## NFR-005: 相容性 (Compatibility)

### 瀏覽器支援
- **主要支援**：Chrome (最近兩個主要版本)、Safari (iOS 15+)。
- **次要支援**：Firefox、Edge。
- **不支援**：Internet Explorer。

### 裝置支援
- PWA 必須在 iOS (Safari) 與 Android (Chrome) 裝置上均可「加入主畫面」並正常運作。
- 介面需採用響應式設計 (Responsive Design)，最小支援寬度 375px (iPhone SE)。

### 生物辨識
- WebAuthn 功能需偵測裝置是否支援，不支援時自動隱藏，不得造成錯誤。

---

## NFR-006: 可維護性 (Maintainability)

### 程式碼品質
- 前後端皆需遵循團隊制定的 Code Style 規範 (Lint / Formatter)。
- 單元測試覆蓋率目標：核心業務邏輯 (金額計算、狀態機轉換) ≥ 80%。

### 日誌與可觀測性
- 所有 API 請求需記錄請求時間、回應狀態碼、處理耗時。
- 關鍵業務操作 (訂單建立/結案/付款/退款) 需寫入業務日誌，含操作者、時間戳與變更前後值。
- 生產環境錯誤需即時通知開發團隊 (如透過 Slack 或 Email Alert)。

### 部署
- 支援 CI/CD 自動化部署流程。
- 資料庫 Schema 變更需透過版本化的 Migration 工具管理。

---

## NFR-007: 離線能力 (Offline Support - PWA)

### 基礎離線支援
- PWA 需註冊 Service Worker，確保在網路短暫中斷時，已載入的頁面仍可瀏覽。
- 保母在服務現場網路收訊不良時，照片上傳應支援排隊機制，待網路恢復後自動重傳。

### 限制
- 離線狀態下不支援訂單建立、付款或狀態變更等需要後端參與的操作。

---

## NFR-008: 法規遵循 (Compliance)

### 個資保護
- 系統需符合台灣《個人資料保護法》之要求。
- 使用者註冊時需明確告知資料蒐集目的與範圍 (已在 PRD-000 設計條款同意機制)。
- 使用者有權要求刪除帳號與相關資料 (待 FR 補充帳號註銷流程)。

### 服務條款
- 平台服務條款與隱私權政策需於使用者首次使用前取得同意。
- 付費方案需另有獨立的付費服務條款 (已在 PRD-012 設計)。

### 金流合規
- 代收代付行為需符合主管機關之相關規範。
- 所有交易紀錄需保存至少 5 年以供查核。

---

## NFR-009: 國際化與在地化 (i18n / L10n)

### 語系
- MVP 階段僅支援繁體中文 (zh-TW)。
- 系統架構應預留多語系擴充能力 (如前端文字抽離為 i18n Key)。

### 時區與貨幣
- 時區：系統固定使用 `Asia/Taipei`。
- 貨幣：統一使用新台幣 (TWD)，金額計算使用 `BigDecimal(18,4)`。

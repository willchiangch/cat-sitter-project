# 專案概述：貓咪保母 PWA (React)
這是一個雙角色 (保母 / 飼主) 的貓咪到府照護 PWA 雛形。
請嚴格按照以下定義的「底部導覽列 (Bottom Navigation)」與對應的「視圖 (Views) 和 動作 (Actions)」進行 React Component 拆分與實作。絕對禁止自行新增、修改或刪減底部導覽列的項目。

## 🌐 全域共用規範與架構限制
1. **身份切換機制**：必須實作 `RoleContext`，透過點擊 Header 右側的「切換/切換角色」按鈕，全域切換 `currentRole` ('sitter' 或 'client')。
2. **顏色主題**：保母端主色系為 Amber (琥珀色)；飼主端主色系為 Blue (藍色)。
3. **佈局限制**：最外層為 `max-w-md mx-auto h-screen relative overflow-hidden` (模擬手機版型)，底部導覽列必須固定於畫面最下方。
4. **狀態管理預留**：此前端專案未來會串接部署於 GCP 的後端與 CloudSQL 資料庫，請確保各 View 的 State (如訂單列表、報價金額、表單輸入) 是獨立且易於抽換為 API Fetch 邏輯的，避免過度耦合。
5. **UI 互動細節**：所有的多行輸入框 (Textarea) 必須實作 `autoResize` (隨打字內容自動延展高度) 的行為；過長的文字段落必須實作 `Text Clamp` (顯示更多/隱藏內容) 的切換邏輯。

---

## 🪟 共用彈出層 (Modals) 規格限制
必須實作以下三種由下方滑出的 Modal (Bottom Sheet 風格，點擊外部可關閉)：

1. **服務方案選擇 Modal (PlanSelectionModal)**：
   * 用於「保母對外預約網頁」更換方案時。
   * 包含單選框 (Radio)、方案名稱、價格、標籤 (如：貓、到府)、以及具備「顯示更多」邏輯的長描述。
   * **防呆邏輯**：必須展示「因日期不符而被反灰停用 (Disabled)」的不可選方案樣式與紅色提示字。
2. **協作編輯紀錄 Modal (EditHistoryModal)**：
   * 用於「毛孩護照」頁面。
   * 包含垂直時間軸 (Timeline)，顯示時間、編輯者(保母或飼主)、以及修改了哪個欄位 (如：更新了環境備註)。
3. **編輯毛孩資料 Modal (PetFormModal)**：
   * 用於飼主端新增/編輯寵物。
   * 包含大頭貼上傳佔位區、欄位包含：名字、寵物種類(Select)、性別(Select)、結紮狀況(Select)、體重(Number 輸入)、以及備註(Textarea)。

## 👩‍⚕️ 模式 A：保母端 (Sitter Mode)
底部導覽列必須「嚴格包含」以下 5 個項目，且僅觸發對應的視圖：

### 1. 行程 (Sitter Dashboard)
* **用途**：檢視今日任務與捷徑。
* **包含動作與元素**：
    * **元素**：顯示「事前問卷已回收」的藍色提示區塊。
    * **動作**：點擊「今日任務/前往訂單/信任圈派單」標籤，可切換對應列表或跳轉視圖。
    * **動作**：點擊任務卡片上的「檢視護照與SOP」，跳轉至【毛孩數位護照】視圖。
    * **動作**：點擊任務卡片上的「任務面板」，跳轉至【任務執行面板】(打卡/上傳照片功能)。

### 2. 訂單 (Sitter Orders)
* **用途**：管理所有生命週期的訂單。
* **包含動作與元素**：
    * **動作**：切換 Tabs (評估中 / 進行中 / 歷史訂單)。
    * **動作 (評估中)**：點擊卡片進入【訂單評估與報價】視圖。該視圖需有「檢視事前問卷答案」區塊、可輸入加價/減價金額的試算器，並包含「確認發送報價」與「拒絕接單」按鈕。
    * **動作 (進行中)**：點擊卡片進入【編輯任務內容】，可拖曳排序 (cursor-move) 或增刪單日專屬 SOP 清單。
    * **動作 (歷史訂單)**：必須包含「月份」與「訂單狀態」的 Select 篩選器、搜尋框，以及動態計算該篩選區間「總收入」的視覺區塊。

### 3. 互助 (Trust Circle)
* **用途**：滿單時將需求轉介給其他保母。
* **包含動作與元素**：
    * **元素**：單選列表 (Radio buttons) 顯示白名單保母 (含狀態與評分)。
    * **動作**：選擇保母後，點擊「確認發送轉介邀請」按鈕 (觸發 Alert 並跳回首頁)。

### 4. 通知 (Sitter Notifications)
* **用途**：系統與訂單狀態通知。
* **包含動作與元素**：
    * **動作**：第一層顯示「飼主列表」，點擊特定飼主後，進入第二層【通知詳細頁】，顯示與該飼主相關的所有訂單動態 (需有返回按鈕)。

### 5. 我的 (Sitter Profile Settings)
* **用途**：保母個人的接單營業設定。
* **包含動作與元素**：
    * **元素**：「專屬接單網址」區塊 (含唯讀 Input 與複製網址按鈕)。
    * **動作**：點擊「預覽客戶預約畫面」，跳轉至隱藏導覽列的【保母對外預約網頁】。
    * **動作**：設定「開放預約期間」(全域日期限制 input)。
    * **動作**：編輯個人簡介 (包含大頭貼、名稱、服務區域、自介 Textarea)。
    * **動作**：新增/編輯服務方案。方案清單支援拖曳排序，每張方案卡包含：公開/隱藏 Switch、價格設定、方案名稱/描述、標籤設定，以及「限制上架與可預約的日期區間」Checkbox。
    * **動作**：事前問卷設定，可新增/拖曳排序/刪除開放式題目。

## 👤 模式 B：飼主端 (Client Mode)
切換為飼主時，底部導覽列必須「嚴格包含」以下 5 個項目：

### 1. 行程 (Client Dashboard)
* **用途**：查看今日保母到府排程。
* **包含動作與元素**：
    * **動作**：點擊今日排程卡片，跳轉至【訂單執行狀況】視圖，檢視保母任務進度。

### 2. 訂單 (Client Orders)
* **用途**：追蹤發出的預約狀態。
* **包含動作與元素**：
    * **動作**：切換 Tabs (評估中 / 進行中 / 歷史紀錄)。
    * **動作 (評估中)**：點擊卡片可查看送單明細，並可點擊「取消送單」。
    * **動作 (進行中)**：點擊卡片進入【訂單執行狀況】。此視圖需包含「協作護照/服務日誌」的入口捷徑、所有排程完成狀態列表 (打勾或時鐘 icon)，當任務全數完成時，點擊「確認任務全部完成 (結案)」。
    * **動作 (歷史紀錄)**：包含月份/狀態篩選器與搜尋框。

### 3. 保母 (Client Sitters)
* **用途**：管理口袋名單與發起預約。
* **包含動作與元素**：
    * **動作**：透過輸入框輸入代碼，點擊「加入」綁定新保母。
    * **動作**：點擊保母卡片 (含頭像、評價、地區) 上的「前往專屬網頁預約」，跳轉至隱藏導覽列的【保母對外預約網頁】。

### 4. 通知 (Client Notifications)
* **用途**：接收保母報價與任務狀態通知。
* **包含動作與元素**：
    * **動作**：第一層顯示「保母列表」，點擊特定保母後，進入第二層詳細通知頁 (需有返回按鈕)。

### 5. 我的 (Client Profile)
* **用途**：管理飼主與毛孩資料。
* **包含動作與元素**：
    * **元素**：「飼主個人資料編輯卡片」(姓名、電話與編輯 Icon)。
    * **動作**：點擊寵物資料卡片旁的編輯 Icon 或「新增寵物卡片」按鈕，開啟 `PetFormModal`。
    * **動作**：最下方的「登出」按鈕。

## 🔗 共用與跨區塊視圖 (觸發時隱藏底部導覽列)
以下視圖不屬於任何單一 Tab，觸發時需覆蓋主畫面，且必須隱藏 Bottom Nav：

1. **毛孩數位護照 (Pet Profile)**：
   * 保母與飼主皆可進入。
   * 包含「純檢視」與「編輯」兩種狀態。
   * 包含醫療與個性備註、家訪環境備註 (皆為 Textarea)。
   * **動作**：必須包含「分享協作 (複製連結)」按鈕，以及開啟 `EditHistoryModal` 的檢視紀錄按鈕。

2. **任務執行面板 (Active Service)** (僅保母端)：
   * **動作**：進入後需有 **Tab 切換**：
     - 「📋 任務檢核表」：顯示該訂單專屬的 SOP Checkbox 清單。
     - 「📸 多媒體日誌」：顯示相機/錄影上傳佔位區塊。
   * **動作**：點擊「任務完成，產出並發送報告」結束任務。

3. **保母對外預約網頁 (Public Booking Page)**：
   * 從保母設定預覽，或從飼主名單發起。
   * **元素**：過長的保母介紹文必須有「顯示更多 / 隱藏內容」的文字收合展開邏輯 (Text Clamp)。
   * **動作 (Step 1 日期與方案)**：必須支援「多個日期群組」的設計 (例如：區間 1 選了 3 天套用 30 分鐘方案；區間 2 選了 1 天套用 60 分鐘方案)，並可獨立刪除群組。點擊更換方案時會叫出 `PlanSelectionModal`。
   * **動作 (Step 2 填寫問卷)**：根據保母設定，動態渲染事前問卷的 Textarea 列表。
   * **動作 (Step 3 價格估算)**：必須根據 Step 1 選擇的「天數」與「方案單價」，動態渲染出費用明細試算表與加總金額，最後點擊「送出預約申請」。
